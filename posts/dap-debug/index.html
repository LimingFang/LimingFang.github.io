Debug in dap-mode for c++ and rust

方里明  收录于  类别 emacs learning
 2022-06-04  2022-06-04  约 831 字   预计阅读 2 分钟 

引言

本文介绍在 emacs 中如何 debug c++、rust。

背景介绍

在 editors/IDE 中为不同语言适配不同 debugger 非常麻烦，因为没有统一的 API，任意一个组合都需要进行适配。 随后微软提出 DAP(debug adapter protocol) 并在 vscode 中使用了如下 debug 通用架构：

  • editor 内部 UI 等组件
  • debug adapter
  • debugger

基本思想是增加一层 DA(debug adapter),DA 和 editors之间通过 DAP 将 debug 相关通讯规范化，例如 “setBreakpoints” 等，其好处是：不用考虑不同 debugger debug 时的设置差异，与 DA 通讯都是规范化的，因此在不同 editor/IDE 之间 DA 可以复用，并且 editor/IDE 端的兼容也能简化。

emacs part

核心是 dap-mode,其在 emacs 端实现 DAP。然后针对不同 DA 提供了不同的 .el 配置文件用于注册 DA。

debug adapter

DA 大部分是为在 vscode 中 debug 开发的，但其与 editors/IDE 其实无关，这部分参考 DAP 实现情况，对于 C++、Rust 情况如下： （前者是 emacs dap-mode 中的适配模块）

  • dap-gdb-lldb for code-debug(both of gdb\lldb)
  • dap-cppptools for vscode-cpptools
  • dap-codelldb for vscode-lldb
  • dap-lldb for lldb-vscode(mirrored)

在选择 DA 时重点考虑 features + 维护情况，c++ 选用 cpptool，rust 选用 codelldb，这俩很不错。

debugger

没啥可谈的，就是 lldb、gdb

setup

install

安装 dap-mode 即可。

config

刚开始摸索的时候，因为 dap-mode doc 不是很多，而且 DA 五花八门，当时也不清楚为什么有这么多 DA 和这么多 emacs 端配置文件，导致花了很久，其实搞清楚逻辑后配置很简单：

  • 下载 DA
  • emacs 端选择选择加载合适的 DA 配置
  • debug 本身进行一些简单配置（例如 UI）
;; basic use
(use-package dap-mode
  :config
;;; use codelldb for c++ and cpptools for c++
  (setq dap-codelldb-download-url "https://github.com/vadimcn/vscode-lldb/releases/download/v1.7.0/codelldb-aarch64-darwin.vsix")
  (setq dap-cpptools-download-url "https://github.com/microsoft/vscode-cpptools/releases/download/v1.9.8/cpptools-osx-arm64.vsix")
  (require 'dap-codelldb)
  (require 'dap-cpptools)
  (dap-codelldb-setup)
  (dap-cpptools-setup)

  :custom
  (dap-auto-configure-mode t                           "Automatically configure dap.")
  (dap-auto-configure-features
   '(sessions locals breakpoints expressions tooltip)  "Remove the button panel in the top.")

template

debug 配置文件概念最初也困扰了很久，只在 vscode 中见到过 launch.json,但其实这些配置文件与 vscode 无关，而是为 DAP 协议服务的，只不过最初 DAP 只在 vscode 中用（且是其提出的），因此很多规范要去 vscode 官网找。

需要注意的是，其中有个 type 属性，填的是用 DA 的标识，例如 gdb 之类的。当初想了很久 DAP 是如何分辨出这对应的是哪个 DA，现在理解是：不同 editor/IDE 对 type->DA 有自己的规定。例如 codelldb 在 emacs 中需要写 codelldb, vscode 中则是 lldb

rust

{
    "configurations": [
	{
	    "type": "codelldb",
	    "request": "launch",
	    "name": "codelldb for rust",
	    "cargo": {
		"args": ["run","--bin"]
	    },
	    "sourceLanguages": ["rust"],
	}
}

c++

{
    "configurations": [
	{
	    "type": "cppdbg",
	    "request": "launch",
	    "name": "cpptools for c++",
	    "program": "${workspaceFolder}/build/${fileBasenameNoExtension}",
	    "args": [],
	    "MIMode": "lldb",
	    "stopAtEntry": "true",
	    "cwd": "${workspaceFolder}",
	    "targetArchitecture": "arm64",
	}
    ]
}
